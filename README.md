# TesterAgent

## 背景

当下测试人员使用 AI 赋能测试的落地场景仍然有限，主要体现为各环节的单点提效，AI 赋能过程中人工主导的比重依然很大。核心痛点在于：测试人员只是把通用 AI 工具拿来使用、再编写一些定制 skill，并没有深入到上下文管理这一层，导致 AI 工具对测试的赋能深度不足，需要人工高频介入。

## 痛点分析

通用 AI 工具采用的是通用的智能体范式与上下文管理机制。在测试赋能这一细分场景下，通用 AI 工具在记忆管理、上下文管理、智能体范式等底层能力上存在天生的不足。仅靠优化 skill 和外置知识库，并不足以落地一套真正智能化的测试系统。

## 解决方案

通过实现测试领域专属的记忆管理机制与上下文管理机制，按需引入混合智能体范式（ReAct + Plan-and-Solve + Reflection），再结合知识库的储备与检索，以 RAG 增强上下文，旨在跑通 AI 赋能测试流程的核心节点：

测试方案 → 测试用例 → 测试执行 → 编写自动化用例

作为探索，只要深度跑通其中 1 个节点，即可验证该方案的可行性。尽管不同节点的 agent 逻辑存在差异，但所沉淀的经验与方法，已足以支撑跑通其余全部节点。

## 更长远的目标

由测试 agent 统筹测试流程各节点的 agent，并负责与产品 agent、研发 agent 通信——以此模拟现实场景中测试人员同样需要频繁与产品、研发沟通核对需求的协作过程。
